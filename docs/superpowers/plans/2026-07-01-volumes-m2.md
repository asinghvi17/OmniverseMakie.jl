# Volumes Milestone-2 (Makie `volume!` with colors + live edits) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `volume!(x, y, z, scalars::Array{Float32,3})` renders a colormap-shaded volume through OmniverseMakie → ovrtx → NVIDIA IndeX, and updates live when the data / colormap / colorrange change.

**Architecture:** A pure-Julia NanoVDB writer (lifted from Hikari) lives in a new `lib/NanoVDBWriter` sub-package. Makie's built-in `Volume` plot routes through the existing diff-node path (`consumed_inputs`/`author_usd_prim!`/`push_to_ovrtx!`): the build writes the dense array to a temp `.nvdb` and calls M1's `author_vdb_volume!`; live edits push minimal writes (colormap → `Colormap` prim attributes; data → re-write + reload). Real transfer-function colors come from the IndeX composite path (spike-gated, verify-or-degrade).

**Tech Stack:** Julia 1.12 · a new `lib/NanoVDBWriter` sub-package (pure Julia, `Zlib_jll` + `GeometryBasics`) · ovrtx RT2 → NVIDIA IndeX (Direct + composite) · USD `UsdVol` schema · Makie `Volume` plot · the existing ComputePipeline diff engine.
